The reproducibility target for the submission should be limited to the blue EPEC solar-PV supply-chain results; the older Lebanese electricity case study can be treated separately or removed later. A clean submission workflow should distinguish between full model reruns, which depend on solver setup and potentially expensive sensitivity runs, and figure/table regeneration from existing Excel outputs.

For the EPEC results, the core artifacts are the planner benchmark (`outputs/llp_planner_results.xlsx`), the player-order sensitivity outputs under `outputs/sens/`, the selected equilibrium run (`sens_ch-row-apac-us-eu-af.xlsx`), and the generated figures/tables used by the IEEE paper. Before submission, it would be useful to add a `REPRODUCE.md` or `reproduce_results.py` that documents the command order for regenerating planner outputs, selected EPEC outputs, welfare calculations, figures, paper image copies, and the final LaTeX build.
